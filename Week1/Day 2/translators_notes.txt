TRANSLATORS IN PROGRAMMING

1. What is a Translator?

A translator is software that converts code written in a programming language into a form that the computer can understand and execute.

Computers understand only machine language (binary).

Translators bridge the gap between human-readable code and machine code.

--------------------------------------------------

2. Types of Translators

There are three main types:

1. Compiler
2. Interpreter
3. Assembler

--------------------------------------------------

3. Compiler

A compiler translates the entire source code into machine code before execution.

General Flow:

Source Code
↓
Compiler
↓
Object Code
↓
Linking
↓
Executable File (.exe)
↓
Execution

Examples:

- GCC
- Clang

Languages Commonly Compiled:

- C
- C++
- Go
- Rust

Advantages:

- Fast execution after compilation
- Errors are reported together

--------------------------------------------------

4. Object Code

Object code is machine-level code produced by a compiler.

Purpose:

- Store translated code
- Allow large programs to be compiled separately
- Used before creating the final executable

Example Extensions:

.obj
.o

--------------------------------------------------

5. Linking

Linking combines:

- Object files
- Required libraries

to create a complete executable program.

Purpose:

- Connect different parts of a program
- Resolve references to functions and variables
- Produce a runnable executable

--------------------------------------------------

6. Interpreter

An interpreter translates and executes code one statement at a time.

General Flow:

Source Code
↓
Interpreter
↓
Output

How It Works:

1. Read instruction
2. Translate instruction
3. Execute instruction
4. Move to next instruction

Advantages:

- Easier debugging
- No separate compilation step

Examples:

- CPython
- Python Interactive Shell
- Node.js

--------------------------------------------------

7. Compiler vs Interpreter

Compiler:

- Translates entire program
- Creates executable file
- Faster execution
- Compilation occurs before execution

Interpreter:

- Translates line by line
- No executable file produced
- Easier debugging
- Translation occurs during execution

--------------------------------------------------

8. Assembler

An assembler converts assembly language into machine code.

Example:

Assembly:

MOV AX, 5

Machine Code:

10111000 00000101

Used in:

- Embedded Systems
- Low-Level Programming

--------------------------------------------------

9. Hybrid Model

Hybrid execution combines features of both compilation and interpretation.

General Flow:

Source Code
↓
Compiler
↓
Bytecode
↓
Virtual Machine
↓
Machine Code Execution

--------------------------------------------------

10. Advantages of Hybrid Model

- Platform Independence
- Better Performance
- Portability
- Easier Distribution
- Runtime Safety Checks

--------------------------------------------------

11. Disadvantages of Hybrid Model

- Requires Virtual Machine
- Higher Memory Usage
- Slower Startup Than Fully Compiled Programs

--------------------------------------------------

12. Python's Hybrid Model

Python does not directly convert source code into machine code.

Python Flow:

Source Code (.py)
↓
Bytecode (.pyc)
↓
Python Virtual Machine (PVM)
↓
Execution

Python uses:

- Compilation to Bytecode
- Interpretation through PVM

Therefore Python follows a Hybrid Execution Model.

--------------------------------------------------

13. Python Virtual Machine (PVM)

PVM stands for Python Virtual Machine.

Responsibilities:

- Read bytecode
- Execute instructions
- Manage program execution

PVM acts as a small virtual computer inside Python.

--------------------------------------------------

14. Key Learnings

- Translators convert code into executable form.
- Main translators are Compiler, Interpreter, and Assembler.
- Compilers translate entire programs.
- Interpreters execute code line by line.
- Assemblers convert assembly language to machine code.
- Python uses a Hybrid Model.
- Python code becomes Bytecode.
- Bytecode runs on the Python Virtual Machine (PVM).
