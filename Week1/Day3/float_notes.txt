FLOAT (float) IN PYTHON

1. What is a Float?

A float (floating-point number) is a numeric data type used to represent numbers that contain a decimal or fractional part.

Examples:

3.14
0.5
-7.25
100.0
0.0

The object has type float, not the variable. :contentReference[oaicite:0]{index=0}

--------------------------------------------------

2. Float Objects

Example:

cgpa = 8.5

Visual:

cgpa ---> 8.5

The variable refers to a float object stored in memory.

--------------------------------------------------

3. Float is Immutable

Float objects cannot be changed after creation.

Example:

x = 2.5
x = 3.5

Incorrect Thinking:

2.5 changed into 3.5

Wrong.

What actually happens:

Before:

x ---> 2.5

After:

x ---> 3.5

The variable starts referring to a different float object.

The original float object remains unchanged. :contentReference[oaicite:1]{index=1}

--------------------------------------------------

4. Common Float Operations

Addition:

2.5 + 1.5

Subtraction:

5.5 - 2.0

Multiplication:

2.5 * 4

Division:

10.0 / 2

Exponentiation:

2.5 ** 2

Floats support all common arithmetic operations. :contentReference[oaicite:2]{index=2}

--------------------------------------------------

5. Why Is It Called Floating Point?

The name comes from scientific notation.

Example:

123.45

can be written as:

1.2345 × 10²

or

12.345 × 10¹

The decimal point appears to "float" to different positions.

This is why the term floating-point number is used. :contentReference[oaicite:3]{index=3}

--------------------------------------------------

6. Float Precision Issue

Example:

0.1 + 0.2

Expected:

0.3

Python Output:

0.30000000000000004

This surprises many beginners. :contentReference[oaicite:4]{index=4}

--------------------------------------------------

7. Why Does This Happen?

Computers store numbers in binary (base 2).

Humans normally use decimal numbers (base 10).

Some decimal values cannot be represented exactly in binary.

Examples:

0.1
0.2

Python stores approximations instead of exact values. :contentReference[oaicite:5]{index=5}

--------------------------------------------------

8. Decimal Analogy

Example:

1/3

becomes:

0.333333333333...

It never ends.

We approximate it:

0.3333

Similarly:

0.1 in binary becomes an infinite repeating value.

Therefore Python stores an approximation. :contentReference[oaicite:6]{index=6}

--------------------------------------------------

9. Why Some Results Look Correct

Example:

0.1 + 0.1

Output:

0.2

Internally there is still a tiny error.

Python hides the error because displaying 0.2 is sufficient to represent the stored float accurately. :contentReference[oaicite:7]{index=7}

--------------------------------------------------

10. Why 0.1 + 0.2 Shows Error

Internally:

0.1 ≈ approximation
0.2 ≈ approximation

When added together:

0.30000000000000004

The approximation error becomes visible.

Python displays the longer value because it more accurately represents the stored float. :contentReference[oaicite:8]{index=8}

--------------------------------------------------

11. Are 0.1 + 0.2 and 0.3 Different?

Example:

0.1 + 0.2

produces one approximation.

0.3

is stored as a slightly different approximation.

Therefore:

0.1 + 0.2 == 0.3

may not always behave as beginners expect. :contentReference[oaicite:9]{index=9}

--------------------------------------------------

12. Why Can't 0.1 Be Stored Exactly?

Humans:

Base 10

Computers:

Base 2

Example:

0.1 in binary becomes:

0.000110011001100110011...

The pattern continues forever.

Memory is finite.

Therefore Python must cut the value off and store an approximation. :contentReference[oaicite:10]{index=10}

--------------------------------------------------

13. Real-Life Analogy

Suppose:

1/3

is stored as:

0.3333

Then:

0.3333 + 0.3333 + 0.3333

gives:

0.9999

instead of:

1

because approximations were used.

Float precision issues occur for the same reason. :contentReference[oaicite:11]{index=11}

--------------------------------------------------

14. Key Learnings

- Floats represent decimal numbers.
- Float objects are immutable.
- Floats use binary representation internally.
- Some decimal numbers cannot be represented exactly.
- Python stores approximations of such values.
- Floating-point arithmetic is approximate, not exact.
- Precision errors are normal and expected.
- The famous example is:

  0.1 + 0.2

  Output:

  0.30000000000000004
